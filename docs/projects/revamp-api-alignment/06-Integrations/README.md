# 06 — Integrations

## Overview

External and infrastructure components that the new backend services depend on: IP-geo provider, realtime transport, caching, observability, and (already-present) payment gateways and CDN.

## Scope

- New: IP-geo provider, Redis cache layer (if not yet provisioned), realtime transport (SSE upgrade or socket gateway), search engine (Atlas Search or alternative).
- Hardening of existing: Stripe / Paystack / Flutterwave webhooks, Cloudinary uploads, Sentry, mailer.

## Dependencies

- **Blocks:** §04 (services can't ship without geo provider, realtime transport, search engine).
- **Blocked by:** §09 (decisions on geo provider, realtime transport, search engine).

## Impacted components

- `binectics-api/src/app.module.ts` (cache config)
- `binectics-api/infra/main.tf` (Redis, possibly search)
- `binectics-api/.env.example` (new vars)
- Netlify configuration on frontend (header forwarding)
- Sentry projects (both repos)
- DNS / domains — no change expected

## Assumptions

- Existing Azure App Service hosts API; same region holds Redis and Mongo.
- Sentry / observability already covers API; we extend coverage.
- Netlify continues to host the frontend.

## Risks

| Risk | Likelihood | Impact | Mitigation |
|---|---|---|---|
| Geo provider rate limits or downtime | M | M | Cache aggressively (EPIC Q); fall back to `Accept-Language` + defaults |
| socket.io sticky sessions don't work on Azure App Service Linux plan | M | H | Choose SSE path or move messaging gateway to a separate plan with sticky |
| Atlas Search not available on current cluster tier | M | M | Use `$text`; document quality limitation |
| Redis added cost without commensurate need | L | L | Cache is justified by aggregate endpoints + search |
| Sentry quota exceeded by SSE error noise | M | M | Sample SSE errors at 10% |
| Webhook signature secrets diverged from payment dashboards | M | H | Rotate + verify in staging before prod |

## Migration & rollout

- Provision new infra first (Redis, search index, geo provider account).
- Update `.env` everywhere, including CI secrets.
- Add health checks before flipping any feature flag in §04.
